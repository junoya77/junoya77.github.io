hexo deploy error,fatal: could not read Username for 'https://github.com'

오랜만에 $ hexo generate; hex deploy를 해보니,
아래와 같은 갑작스러운 에러가 발생했습니다.

fatal: could not read Username for 키워드로 구글링을 해보니,
아래와 같은 링크들이 검색되었고,
https://github.com/hexojs/hexo-deployer-git/issues/71
https://hanumoka.github.io/2018/03/29/error-hexo-deploy/

github.com 대상 ssh 접속 환경 설정 및 _config.yml 설정파일 수정이 필요하다는 내용이었습니다.

그러나, 이전에 잘 되던 것이 갑자기 안되는 것이 이상한데다,
ssh로 세팅하는 건 너무 많은(또는 귀찮은) 작업이 발생하는 것 같아,
좀 더 간단한 솔루션이 없는 지 추가로 구글링 하다보니, 아래 링크를 발견했습니다.
https://github.com/hexojs/hexo/issues/3052

여기서는, 아래와 같이 한 줄의 command 실행만으로 해결할 수 있다고 하여
그대로 해 보았더니, 정말 해결되어 $ hexo deploy가 정상 동작하였습니다.

git config는 git 관련 옵션을 설정하겠다는 의미이고,

--global은 ~/.gitconfig 설정파일에 대한 read/write를 하겠다라는 의미이고,

credential.helper는 username/password credential이 필요한 경우
이를 지원하기 위해 호출될 helper를 지정하는 옵션으로,
wincred로 지정한 것은, windows credential manager를 사용하겠다는 의미입니다.

  • 참고
    https://git-scm.com/docs/git-config
    https://stackoverflow.com/questions/38333752/trying-to-understand-wincred-with-git-for-windows-confused?utm_medium=organic&utm_source=google_rich_qa&utm_campaign=google_rich_qa

단지 이런 수정이 위 문제를 해결한 건지를 확인하기 위해,
아래와 같이 ~/.gitconfig 파일의 “credential” 부분을 삭제해 보았습니다.

그리고나서 다시 $ hexo deploy를 실행한 결과, 다시 에러가 발생함을 확인할 수 있었습니다.

다시 $ git config --global credential.helper wincred를 실행하여
~/.gitconfig 파일에 credential 설정을 복원시키니
hexo deploy가 정상 동작하였습니다.

그런데, 아래 링크를 보니, 갑자기 이런 현상이 발생한 원인은
github 측에서 2018년 2월 22일부터 weak encryption 에 대한 지원을 중단하면서이고,
git을 2.16.0 버전으로 업데이트하여 해결하라는 가이드를 주고 있습니다.
https://github.com/Microsoft/Git-Credential-Manager-for-Windows

git for windows 업데이트 사이트에 가보니, 현재 버전은 2.17.0 이어서,
Git-2.17.0-64-bit.exe 파일로 설치해 보았습니다.
https://github.com/git-for-windows/git/releases/tag/v2.17.0.windows.1

~/.gitconfig 파일의 내용을 보니, 2.17.0으로 업데이트 하기 전의
credential 설정이 여전이 남아 있는 듯 하여, credential 설정을 삭제한 후,
$ hexo generate; hexo deploy를 실행한 결과, 깔끔하게 잘 실행되었습니다.

그래서, 결론은,
ssh 설정, git config --global credential.helper wincred와 같은 수정이 아닌,

그냥 git을 업데이트하면 되는 이슈였고,
(원인 github 서버 쪽에서 weak encryption 지원 중단으로 인한 것으로 보임)
git 업데이트를 자주 해야 겠네요..
