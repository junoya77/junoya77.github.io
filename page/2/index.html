torrent file format 분석해 보기

BitTorrent에서 사용하는 ~.torrent 파일의 구조가 어떻게 되어 있는지 확인해 보겠습니다.

  • 참고 : https://en.wikipedia.org/wiki/Torrent_file

~.torrent 파일은 bencode 방식의 구조로 구성되어 있기에,
먼저, bencode에 대해 알아봅니다.

bencode란,
BitTorrent의 ~.torrent 파일에서 metadata를 기술하기 위해 사용하는
encoding 방법을 말합니다.

  • 참고 : https://en.wikipedia.org/wiki/Bencode

정수는 “i<숫자>e”로 표현됩니다.

  • 42 => i42e
  • 0 => i0e
  • -42 => i-42e

string은 “<length>:<contents>”로 표현됩니다.

  • spam 이라는 string => 4:spam

list는 “l<contents>e”로 표현됩니다.

  • spam과 42로 구성된 list => l4:spami42ee

dictionary는 “d<contents>e”로 표현됩니다.

  • {“bar”:”spam”, “foo”:42} 인 경우, => d3:bar4:spam3:fooi42ee

~.torrent 파일은,
bencode 방식으로 인코딩 된 dictionary로 구성되어 있습니다.
dictionary이기 때문에, key:value로 표현되며,
아래와 같은 key를 가지고 있습니다.

  • announce :

    • tracker의 URL 정보를 가집니다..
      • tracker란, 파일 전송을 하고 있는 seed peer를
        client가 잘 찾을 수 있도록 도와주는 서버를 말합니다.

  • announce-list :

    • 기존 announce에서는 하나의 tracker URL만 기술할 수 있어서,
    • 여러 tracker를 기술하기 위한 dictionary key로 추가되었습니다.

  • info:

    • files:
      • 공유되는 파일이 여러 개일 경우 사용됩니다.
      • dictionary의 list 형태이며,
      • 각 dictionary가 하나의 파일에 관한 정보을 가지고 있습니다.
      • dictionary는 다음의 key를 가집니다.
        • length:
          • byte 단위의 파일 크기
        • path:
          • subdirectory명 및 파일명으로 구성하는 string list입니다.
    • length:
      • byte 단위의 파일 크기를 의미합니다. 공유되는 파일이 1개일 때만 사용됩니다.
    • name:
      • 공유 파일이 1개일 경우, 파일이 저장될 때 파일명입니다.
      • 공유 파일이 여러 개일 경우, 파일들이 저장될 디렉토리명입니다.
    • piece length:
      • piece의 byte 단위 크기를 의미합니다.
      • piece란 공유되고 있는 컨텐츠를 구성하는 하나의 조각을 의미합니다.
      • 보통 262,144 B = 256 KB = 2^8 KB 입니다.
    • pieces:
      • 각각의 piece들을 SHA-1 해시처리한 해시값들을 모아놓은 hash list 입니다.

샘플 ~.torrent 파일 내용을 한번 보겠습니다.

torrent 공유 사이트에서 ~.torrent 파일을 하나 내려 받아 열어봅니다.
hex viewer로 열어봅니다.

이어서 나오는 8은 길이 8의 string을 의미하고,
이어서 8글자 announce가 나옵니다.

announce는 tracker의 URL 정보를 가지는 dictionary key 입니다.

이어서 32 string length가 나오고,
