what is ethereum?

  • 참고
    • https://ethereumdev.io/what-is-ethereum/

위 Ethereum 설명 페이지에 대한 내용을 학습해 봅니다.

Ethereum은, 블록체인 기술에 기반한 “open software platform”이라고 합니다.

Ethereum의 블록체인은 아래와 같이 2가지 유형의 object를 포함한다고 합니다.

  1. wallet

    • Ethereum을 보유하기 위해 사용되는 object입니다.

  2. smart contract

    • Ethereum을 보유한다는 측면에선 wallet과 유사하지만,
    • 코드로 만들어진다는 점이 다릅니다.
    • 이 코드는 블록체인에 배포되어,
    • 네트워크에 의해 실행됩니다.

어떤 거래(transaction)를 발생시키면,
이 ‘거래’는 블록체인으로 제출되고,
채굴자(miner)가 이 ‘거래’를 이제 막 채굴된 따끈따끈한 블록으로 통합하는 순간,
‘거래’가 실행됩니다.
채굴자(miner)는 이 채굴 활동의 대가로 약간의 수수료를 받는데,
이 수수료를 “gas price” 라고 합니다.

어떤 ‘거래’를 ‘smart contract’로 발생시키면,
‘smart contract’의 코드가 시작됩니다.

‘smart contract’의 코드도 사람이 만든거라, 버그가 있을 수 있으며,
‘분산’의 특성 때문에 이 버그는 쉽게 고치기 어렵습니다.

pandas.get_dummies()와 one-hot-encoding의 차이점 (수정중)

pandas.get_dummies() 함수는 categorical variable을 dummy variable로 변환합니다.

  • 참고 : https://pandas.pydata.org/pandas-docs/stable/generated/pandas.get_dummies.html

그러면, dummy variable 이란 무엇일까요?

dummy variable은
indicator variable, design variable, boolean indicator,
binary variable, qualitative variable 이라고도 부른다고 합니다.

dummy variable 이란,
숫자만을 다루는 regression model이
categorical value도 숫자로 처리할 수 있도록 해주기 위해,
categorical value의 각 값들에 대한 presence, absence를 1, 0으로 표시하는 방식으로
변환한 결과로 나타난 variable입니다.

예를 들어,
어떤 주택 매매가에 대한 regression model을 만들 때,
집이 동쪽(E)에 있는지, 동남쪽(SE)에 있는지, 북서쪽(NW)에 있는지 여부가
매매가에 영향을 준다고 했을 때,
(단, 이때, E, SE, NW 값은 categorical value이며 mutually exclusive 합니다.)

일단, 아래와 같이 ‘E’, ‘SE’, ‘NW’라는 3가지 값을 가지는 category variable을 만들어 봅니다.

이제, loc category variable으로부터
pandas.get_dummies() 함수를 이용하여 dummy variable을 만들어 봅니다.

위에서 보면, ‘E’, ‘NW’, ‘SE’의 경우에 대해
presense는 ‘1’, absense는 ‘0’으로 표시되었음을 알 수 있습니다.

이렇게 categorical variable을 dummy variable화 하면,
숫자만을 다루는 regression model에서도
categorical variable의 영향을 반영하여 modeling이 가능해집니다.

  • 참고 : https://www.moresteam.com/whitepapers/download/dummy-variables.pdf

그런데, “dummy variable trap”이란게 있다고 합니다.

  • 참고 : http://facweb.cs.depaul.edu/sjost/csc423/documents/dummy-variable-trap.htm

그리고, one-hot-encoding과 dummry variable의 차이에 대해서도 확인이 필요합니다…

categorical data type 알아보기

아래 링크의 내용을 나름대로 요약 정리해 보았습니다.

  • 참고 : https://pandas.pydata.org/pandas-docs/stable/categorical.html

고정(제한)된 갯수의 값 중에 하나만을 가질 수 있는 변수(variable)의 data type이
categorical data type이 됩니다.

여기서, “변수(variable)”라 함은
2차원 data table에서의 “컬럼(column)”으로 보면 될 것 같습니다.

즉, 해당 “컬럼”에 나타나는 data들의 값들이 연산이 가능한 숫자들이 아닌,
어떤 의미를 가지는 제한된(고정된) 값들 중 하나가 됩니다.

예를 들면, 성별, 혈액형, likert scale 형태의 rating 등이 categorical value가 됩니다.

likert scale이란,
설문 조사 때 많이 보는, “매우나쁨, 나쁨, 보통, 좋은, 매우좋음” 스타일의 scale을 말합니다.

아래와 같이 pandas.Series의 dtype option을 category로 줌으로써,
category value로 구성된 어떤 array를 categorical data type으로 변환할 수 있습니다.

이때, categorical value들의 값들은 코드로 저장되며,

각 categorical value들이 어떤 코드로 변환되었는지는,
pandas.Series.cat.categories를 통해 확인할 수 있습니다.

위에서 보면, Index([...]) 안에서 category value들이 list로 있는데,
list의 index가 해당 category value의 코드가 됩니다.

즉, bad’(0), ‘good’(1)’, ‘soso’(2), ‘very bad’(3), ‘very good’(4)로
코드가 할당되었습니다.

category variable을 category value가 아닌 코드로 보고 싶다면,
pandas.Series.cat.codes를 통해 확인할 수 있습니다.

그런데, 코드들이 likert scale 순서대로
‘very good’(0), ‘good’(1), ‘soso’(2), ‘bad’(3), ‘very bad’(4) 순이 되면 좋았을텐데,
‘bad’(0), ‘good’(1)’, ‘soso’(2), ‘very bad’(3), ‘very good’(4)로 되어 있어
categorical value들의 순서가 이상하게 되어 버렸습니다.

원하는 순서대로 코드 재조정을 하려면,
reorder_categories() 함수를 사용하면 됩니다.

reorder_categories() 함수 호출 시,
inplace=True 로 해주어야 a1의 내의 코드가 재부여 됩니다.

inplace option의 default 값은 False로,
False인 경우는, 현재 다루어지고 있는 category variable data 자체는 변경하지 않고,
변경이 적용된 복사본을 return 한다고 합니다.

  • 참고 : https://pandas.pydata.org/pandas-docs/stable/generated/pandas.Series.cat.reorder_categories.html

통계학에서의 categorical variable은 “순서(order)”를 가질 수 없다고 하는데,
pandas의 categorical data type은 “순서(order)”를 가질 수 있다고 합니다.
categorical value에 매핑되는 코드를 통해 순서가 정해지는 것 같습니다.

categorical data type은 당연히, 덧셈, 뺄셈과 같은 numerical operation은 할 수 없다고 합니다.
“남성” + “여성” 은 의미가 없습니다.

categorical value는 categorical variable이 가질 수 있는 제한된(고정된) 값들 중 어느 하나이거나,
np.nan (Not A Number) 값이어야 합니다.

NaN(Not A Number)이란, number type이긴 하지만,
수학적 정의에 의해 “undefined value” 또는 “unrepresentable value”인 경우를 표현 합니다.

예를 들어, 0/0은 real number로 표현할 수 없기에 “NaN”가 됩니다.
또한 sqrt(음수)는 허수이기에 floating-point number로 표현할 수 없어, 이 또한 “NaN”이 됩니다.

numpy의 np.nan으로 NaN을 표현합니다.

np.nan의 type을 보면 <class 'float'>임을 알 수 있습니다.

NaN과 그 외 다른 모든 것들과의 비교 연산 결과는 모두 False가 됩니다.
단, “NaN != NaN”은 True 입니다.

categorical data type의 내부 구조는 아래와 같이 2개의 array로 구성되었다고 합니다.

1) categories array
2) integer array of codes

categorical data type을 사용하는 경우,
string array 대비 memory 사용을 절약할 수 있다고 합니다.

string array의 경우,
동일한 string이 여러번 반복될 경우, 반복되는 경우마다 string 값이 메모리를 차지하지만,

categorical data type의 경우 string의 값이 code화 되어 있기에,
동일한 string 값이 여러번 반복되더라도
string 값 그자체가 반복해서 메모리를 점유하는 것이 아닌,
string 값보다 더 size가 작은 code 값이 메모리를 점유하기에
메모리 사용에 있어 string array보다 이점이 생기게 됩니다.

아래 실행 결과를 보면,
string array의 size는 84-byte 이지만,
categorical value로 구성된 pandas.Series는 31-byte인 것을 알 수 있습니다.

md5 collision을 이용한 듯 한데, 세부적인 내용은 잘 이해가 안가네요.

그것보단 앞서,
이미지 형태로 된 hex 값들로 구성된 hash 값이 PDF 내 포함되어 있는데,
각 hex 값 이미지를 간단히 한 줄 명령어로 만들어 보는 방법을 정리해 보았습니다.

먼저 command line 상에서 convert를 사용하면,
text 값을 바로 jpg 이미지 파일로 만들 수 있습니다.

아래는 숫자 1을 -annotate 옵션으로 입력 받아
횐 배경에 검은색 글씨로 1이라고 써진 정사각형 1.jpg 이미지를 생성하는 명령입니다.

-size는 jpg 이미지 크기를 지정하고,
xc:white는 이미지의 배경을 흰색으로 하겠다는 의미이며,
-pointsize는 펜굵기,
-fill black은 펜 색깔을 검은색으로 하겠다는 것이고,
-annotate +20+80 1은 20,80 위치에 1 이라는 text를 주석으로 write 하겠다는 의미입니다.

위 명령을 1 ~ F 까지 값들에 각각 적용하기 위해,
1 ~ F 값들을 출력하는 명령어를 작성해 봅니다.

printf의 %x 값이 소문자로 출력되어서 이를 대문자로 변환하기 위해 tr을 사용했습니다.

1 ~ F 까지 출력하는 output을 가지고, for loop 처리가 가능한지 테스트 해 봅니다.

이제 위 두가지 명령어를 조합하여 1 ~ F 숫자를 text로 입력받아
각각을 jpg 이미지 파일로 생성합니다.
