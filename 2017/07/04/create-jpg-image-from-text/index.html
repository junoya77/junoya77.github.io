한줄 명령어로 hex 값 jpg 이미지 파일 만들기

POC||GTFO 0x14 내용을 보면,
해당 PDF의 md5 hash값이 PDF 문서 내 이미지로 포함되어 있는 것을 볼 수 있습니다.

  • 참조 : https://www.alchemistowl.org/pocorgtfo/pocorgtfo14.pdf

md5 collision을 이용한 듯 한데, 세부적인 내용은 잘 이해가 안가네요.

그것보단 앞서,
이미지 형태로 된 hex 값들로 구성된 hash 값이 PDF 내 포함되어 있는데,
각 hex 값 이미지를 간단히 한 줄 명령어로 만들어 보는 방법을 정리해 보았습니다.

먼저 command line 상에서 convert를 사용하면,
text 값을 바로 jpg 이미지 파일로 만들 수 있습니다.

아래는 숫자 1을 -annotate 옵션으로 입력 받아
횐 배경에 검은색 글씨로 1이라고 써진 정사각형 1.jpg 이미지를 생성하는 명령입니다.

-size는 jpg 이미지 크기를 지정하고,
xc:white는 이미지의 배경을 흰색으로 하겠다는 의미이며,
-pointsize는 펜굵기,
-fill black은 펜 색깔을 검은색으로 하겠다는 것이고,
-annotate +20+80 1은 20,80 위치에 1 이라는 text를 주석으로 write 하겠다는 의미입니다.

위 명령을 1 ~ F 까지 값들에 각각 적용하기 위해,
1 ~ F 값들을 출력하는 명령어를 작성해 봅니다.

printf의 %x 값이 소문자로 출력되어서 이를 대문자로 변환하기 위해 tr을 사용했습니다.

1 ~ F 까지 출력하는 output을 가지고, for loop 처리가 가능한지 테스트 해 봅니다.

이제 위 두가지 명령어를 조합하여 1 ~ F 숫자를 text로 입력받아
각각을 jpg 이미지 파일로 생성합니다.
