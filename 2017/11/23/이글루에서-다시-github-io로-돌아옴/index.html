기타

이글루에서 다시 github.io로 돌아옴

다른 이에게 아주 쉽게 알려주도록 노력하는 것은
효과적인 학습 방법 중 하나입니다.

다른 이에게 내용을 알려주기 위해
먼저 자신이 해당 내용을 잘 이해하고 있어야 하고,

그 내용을 쉽게 전달하기 위해
그 내용의 핵심을 이해하고 있어야 합니다.

쉽게 알려주고자 하는 노력의 과정에서
오히려 그 주제에 대한 자신의 이해도가 높아지게 됩니다.

그렇다면 쉽게 알려주어야 할 대상,
즉, 내 얘기를 잘 들어주고,
가끔씩 예상치 못한 질문도 해줄 수 있는 상대방이 필요한데,

다른 이의 말을 잘 들어주는 사람은 찾기가 어렵습니다.
그래서, 이를 대체할만한 수단으로 블로깅이 있습니다.

마치 누군가 내 얘기를 듣고 있다고 가정하고,
그 사람에게 어떤 주제를 쉽게 알려주는 형태로
블로그에 글을 작성한다면,
그에 상응하는 나의 지력 상승의 효과를 거둘 수 있을 것 같습니다.

게다가 말로 하는 것보다는 글로 쓰는 것이
앞 뒤 맥락에 대한 퇴고 과정을 통해
좀 더 깊이있는 사고의 과정을 경험을 할 수 있을 것 같습니다.

이와 같은 블로깅의 효과를 누리고자
github.io를 이용해 보았는데,
github.io는 글을 관리할 수 있는 환경을 별도로 제공해 주지 않습니다.

github.io에서 글을 관리하기 위해서는,
별도 framework을 자신의 PC에 설치/관리해야 합니다.

저는 hexo 툴을 사용해 보았습니다.

그러나 이런 framework을 설정하는데 들어가는 노력이 아까워
egloos.com에선 좀 더 편하게 블로깅을 할 수 있지 않을까 하여
약 이틀간 egloos.com에서 글을 작성해 보았습니다.

egloos.com은 이미 10년 전에 작성했던 글들도 있어
웬지 고향에 돌아온 느낌도 들었습니다.

그러나,
code syntax highlighting 이라던가,
“..” 등 특수문자가 포함된 내용은 포스팅 시 오류가 발생했고,
PC 화면에 잘 보이던 내용이 모바일 화면에선 폰트 차이가 발생하는 등
예상했던 것만큼 편하지 않아,

결국 github.io에서 framework 설정을 가다듬는게
egloos.com에서 잡다한 문제로 시간을 낭비하는 것보다 더 나을 것 같았습니다.

그래서, github.io에서 hexo 툴로 글을 작성하는 방식을 취하기로 하였습니다.
위키처럼 이러저런한 학습 고민/정리 내용들을 위키처럼 정리해 보기로 했습니다.
