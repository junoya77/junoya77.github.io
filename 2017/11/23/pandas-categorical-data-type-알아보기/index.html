categorical data type 알아보기

아래 링크의 내용을 나름대로 요약 정리해 보았습니다.

  • 참고 : https://pandas.pydata.org/pandas-docs/stable/categorical.html

고정(제한)된 갯수의 값 중에 하나만을 가질 수 있는 변수(variable)의 data type이
categorical data type이 됩니다.

여기서, “변수(variable)”라 함은
2차원 data table에서의 “컬럼(column)”으로 보면 될 것 같습니다.

즉, 해당 “컬럼”에 나타나는 data들의 값들이 연산이 가능한 숫자들이 아닌,
어떤 의미를 가지는 제한된(고정된) 값들 중 하나가 됩니다.

예를 들면, 성별, 혈액형, likert scale 형태의 rating 등이 categorical value가 됩니다.

likert scale이란,
설문 조사 때 많이 보는, “매우나쁨, 나쁨, 보통, 좋은, 매우좋음” 스타일의 scale을 말합니다.

아래와 같이 pandas.Series의 dtype option을 category로 줌으로써,
category value로 구성된 어떤 array를 categorical data type으로 변환할 수 있습니다.

이때, categorical value들의 값들은 코드로 저장되며,

각 categorical value들이 어떤 코드로 변환되었는지는,
pandas.Series.cat.categories를 통해 확인할 수 있습니다.

위에서 보면, Index([...]) 안에서 category value들이 list로 있는데,
list의 index가 해당 category value의 코드가 됩니다.

즉, bad’(0), ‘good’(1)’, ‘soso’(2), ‘very bad’(3), ‘very good’(4)로
코드가 할당되었습니다.

category variable을 category value가 아닌 코드로 보고 싶다면,
pandas.Series.cat.codes를 통해 확인할 수 있습니다.

그런데, 코드들이 likert scale 순서대로
‘very good’(0), ‘good’(1), ‘soso’(2), ‘bad’(3), ‘very bad’(4) 순이 되면 좋았을텐데,
‘bad’(0), ‘good’(1)’, ‘soso’(2), ‘very bad’(3), ‘very good’(4)로 되어 있어
categorical value들의 순서가 이상하게 되어 버렸습니다.

원하는 순서대로 코드 재조정을 하려면,
reorder_categories() 함수를 사용하면 됩니다.

reorder_categories() 함수 호출 시,
inplace=True 로 해주어야 a1의 내의 코드가 재부여 됩니다.

inplace option의 default 값은 False로,
False인 경우는, 현재 다루어지고 있는 category variable data 자체는 변경하지 않고,
변경이 적용된 복사본을 return 한다고 합니다.

  • 참고 : https://pandas.pydata.org/pandas-docs/stable/generated/pandas.Series.cat.reorder_categories.html

통계학에서의 categorical variable은 “순서(order)”를 가질 수 없다고 하는데,
pandas의 categorical data type은 “순서(order)”를 가질 수 있다고 합니다.
categorical value에 매핑되는 코드를 통해 순서가 정해지는 것 같습니다.

categorical data type은 당연히, 덧셈, 뺄셈과 같은 numerical operation은 할 수 없다고 합니다.
“남성” + “여성” 은 의미가 없습니다.

categorical value는 categorical variable이 가질 수 있는 제한된(고정된) 값들 중 어느 하나이거나,
np.nan (Not A Number) 값이어야 합니다.

NaN(Not A Number)이란, number type이긴 하지만,
수학적 정의에 의해 “undefined value” 또는 “unrepresentable value”인 경우를 표현 합니다.

예를 들어, 0/0은 real number로 표현할 수 없기에 “NaN”가 됩니다.
또한 sqrt(음수)는 허수이기에 floating-point number로 표현할 수 없어, 이 또한 “NaN”이 됩니다.

numpy의 np.nan으로 NaN을 표현합니다.

np.nan의 type을 보면 <class 'float'>임을 알 수 있습니다.

NaN과 그 외 다른 모든 것들과의 비교 연산 결과는 모두 False가 됩니다.
단, “NaN != NaN”은 True 입니다.

categorical data type의 내부 구조는 아래와 같이 2개의 array로 구성되었다고 합니다.

1) categories array
2) integer array of codes

categorical data type을 사용하는 경우,
string array 대비 memory 사용을 절약할 수 있다고 합니다.

string array의 경우,
동일한 string이 여러번 반복될 경우, 반복되는 경우마다 string 값이 메모리를 차지하지만,

categorical data type의 경우 string의 값이 code화 되어 있기에,
동일한 string 값이 여러번 반복되더라도
string 값 그자체가 반복해서 메모리를 점유하는 것이 아닌,
string 값보다 더 size가 작은 code 값이 메모리를 점유하기에
메모리 사용에 있어 string array보다 이점이 생기게 됩니다.

아래 실행 결과를 보면,
string array의 size는 84-byte 이지만,
categorical value로 구성된 pandas.Series는 31-byte인 것을 알 수 있습니다.
